# 仕様書
## Free Market Data Lab 実装仕様 v0.1

### 1. 目的

既存システムに対して、米国株の監視・検索・詳細閲覧を行うための新規ページ **Free Market Data Lab** を追加する。
本ページは、既存の Monitor/ML/Compare/Strategy とは独立した導線で提供し、次の方針を満たす。

- **初見ユーザーでも操作が分かる**
- **無料で運用できる**
- **既存ページを壊さず、新しいページとして追加する**

---

## 2. 基本方針

### 2.1 実装方針
本機能は既存の `/` ページを改修せず、**新規ページ `/market-data-lab`** として追加する。
既存の `MarketDataHub`、`symbol catalog`、`historical`、`security overview`、`stream` などは可能な限り再利用する。

### 2.2 UX方針
本ページは「高機能ページ」ではなく、**初見の利用開始障壁を下げるページ**として設計する。
そのため、既存 Monitor よりも以下を強化する。

- 初回表示時の3ステップ案内
- 各重要項目の `?` ヘルプ
- 何を押せばよいかが分かる空状態
- 無料プラン制約の見える化
- エラー時の復旧案内

### 2.3 無料方針
必須構成は **無料プランのみ** で完結させる。

- 標準動作: **FMP Free**
- 任意拡張: **Twelve Data Basic**
- ローカルコメント生成機能は**初期リリースでは無効化またはオプション扱い**
- 有料 API を前提にした UI は出さない

---

### 3.1 新規追加ページ
- ルート: `/market-data-lab`
- ナビゲーション名: `Market Data`

### 3.2 追加対象ファイル

  - `app/api/market.py` に bootstrap 用 API を追加
  - `app/stores/` に onboarding 状態保存用 store を追加

---

## 4. ページの役割

本ページは次の4機能に絞る。

1. **銘柄検索**
2. **ウォッチリスト管理**
3. **リアルタイムまたは準リアルタイム監視**
4. **銘柄詳細閲覧**

ML 実験、比較、戦略評価は本ページでは扱わない。
その導線は既存の `/ml-lab`、`/compare-lab`、`/strategy-lab` に任せる。

---

## 5. 対象ユーザー

### 5.1 主対象
- 株価監視ページを初めて使うユーザー
- 米国株の基本的な値動きを見たいユーザー
- まず無料で試したいユーザー

### 5.2 想定リテラシー
- ティッカーの意味が曖昧でも利用できる
- API 制限を知らなくても利用できる
- 「現在値」「前日比」「出来高」程度の言葉なら理解できる

---

## 6. 画面仕様

## 6.1 全体構成
ページは以下の3カラムではなく、**初見向けに2段構成**とする。

### 上段
- ページタイトル
- 無料プラン説明バナー
- 初回ガイド
- プロバイダー状態表示

### 下段 左
- 銘柄検索
- ウォッチリスト
- 利用制約表示

### 下段 右
- 選択銘柄の詳細
- チャート
- KPI
- 補足説明

---

## 6.2 初回ガイド
ページ上部に常時または初回のみ表示する。

表示内容:
- Step 1: 銘柄を検索
- Step 2: ウォッチリストへ追加
- Step 3: 右側で詳細を見る

要件:
- 初回表示時は展開状態
- ユーザーが閉じたらローカル保存またはサーバ保存
- 再表示ボタンあり
- 各 step に短い説明と例を付ける

例:
- `AAPL`, `MSFT`, `NVDA` などを検索できます
- 追加できるのは無料モードでは最大8銘柄です
- 銘柄をクリックすると右側に詳細が表示されます

---

## 6.3 ヘルプマーク仕様
初見向けのため、以下のすべてに `?` ヘルプを付与する。

- データプロバイダー
- 更新モード
- 前日比
- 出来高
- VWAP
- MA
- ATR
- ベータ / 相関
- キャッシュ
- クレジット
- フォールバック

### ヘルプ表示形式
- PC: hover + click
- モバイル: click
- 実装は `ml_lab.html` で使っている `details/summary` パターンを流用

### ヘルプ文の要件
- 1文目で「何か」
- 2文目で「どう見ればよいか」
- 3文目で「無料版では何が制限されるか」

---

## 6.4 無料プラン説明バナー
上部に固定表示する。

表示項目:
- 現在のプロバイダー: `FMP Free` / `Twelve Data Basic` / `Both`
- 無料モードの上限
- 現在のウォッチ銘柄数
- REST 利用時の更新間隔目安
- 一部項目は未対応であること

文言例:
- 「このページは無料プラン前提です」
- 「最大8銘柄まで監視できます」
- 「一部の企業イベント・ニュース・板情報は表示されません」

---

## 6.5 銘柄検索
### 入力仕様
- テキスト入力
- シンボル前方一致 + 社名部分一致
- 候補 dropdown 表示
- 1回の追加は1銘柄ずつ

### 表示項目
- Symbol
- Company Name
- Exchange
- Country
- Provider source

### 操作
- Enter で先頭候補選択
- クリックでウォッチリスト追加
- 上限超過時は追加不可

### エラー
- 候補なし
- API 取得失敗
- 無料枠上限超過

---

## 6.6 ウォッチリスト
### 表示項目
- Symbol
- 現在値
- 前日比%
- 更新時刻
- 取得元
- 削除ボタン

### 操作
- クリックで詳細表示
- ドラッグ並び替えは初版では不要
- ソートは `symbol / price / changePct` のみ

### 制約
- 無料モード上限: 8銘柄
- 初期表示は既存設定を引き継いでよい
- 新ページ専用のウォッチリスト状態を持ってもよい

---

## 6.7 銘柄詳細パネル
### 表示項目
- 現在値
- 前日比（値幅・%）
- 当日高値/安値
- 出来高
- 売買代金
- MA20 / MA50
- ATR14
- ギャップ
- β / 相関（対 SPY、可能なら QQQ）
- 更新時刻
- データソース

### チャート
- 1分
- 5分
- 日足
- レンジ切替: `1D / 5D / 1M / 6M / 1Y / MAX`

### 補足表示
無料で未取得の項目は削除せず、**`未対応`** として理由を表示する。

例:
- 板情報: 無料プロバイダー未対応
- ニュース: 本ページ対象外
- 企業イベント: 無料モードでは簡易表示のみ

---

## 6.8 空状態
初見向けなので、空状態文言は必須。

### ウォッチリスト空状態
- 「左上の検索欄から銘柄を追加してください」
- 例銘柄ボタン: `AAPL`, `MSFT`, `NVDA`

### 詳細空状態
- 「ウォッチリストから銘柄を選ぶと、ここに詳細を表示します」

## 7. データソース仕様

## 7.1 必須要件
本機能は **無料のみで成立** しなければならない。

### 標準構成
- 基本価格・銘柄検索・履歴: **FMP Free**
- 追加オプション: **Twelve Data Basic**

### 方針
- API キー未設定でも、UI 上で何が不足しているかを明示する
- 片方しか設定されていなくてもページは動作する
- `both` は拡張モードであり、必須ではない

---

## 7.2 優先順位
### 推奨
1. `both`
2. `fmp`
3. `twelvedata`

### 理由
- FMP Free だけでも最低限のページ成立性が高い
- Twelve Data は WebSocket 面で利点があるが、無料制約が厳しい
- 初見向けページは「止まらないこと」を優先する

---

## 7.3 未対応項目
以下は初版では非必須。

- 板情報
- ニュースフィード
- 決算カレンダー統合
- 高度な企業イベント
- 自動要約 LLM

## 10. 初見ユーザー向け UX 要件

## 10.1 ガイド優先
このページでは「機能量」より「理解しやすさ」を優先する。

### 必須要件
- 各セクションに1行説明
- 難しい用語に `?`
- 失敗時に次の操作を案内
- 初回導線をページ先頭に配置

---

## 10.2 文言ルール
文言は短くする。

悪い例:
- `Fallback activated due to degraded websocket heartbeat condition`

良い例:
- `リアルタイム接続が不安定なため、定期更新に切り替えました`

---

## 10.3 エラー表示
エラーは3段階で出す。

- 情報: 更新待ち
- 注意: 一部項目未取得
- 警告: API キー未設定 / 上限超過 / 取得失敗

各エラーに復旧案内を付ける。

例:
- 「無料枠上限に達しました。監視銘柄を減らすか、更新間隔を延ばしてください」

---

## 11. 非機能要件

## 11.1 パフォーマンス
- 初回描画: 3秒以内目標
- bootstrap API: 1秒以内目標
- ウォッチリスト更新: 1秒以内目標
- 詳細更新: 3秒以内目標

## 11.2 可用性
- WebSocket 不可でも REST だけで成立
- 一部 API 失敗時もページ全体は壊さない
- 詳細の一部欠損は許容し、未対応表示に落とす

## 11.3 セキュリティ
- API キーはサーバー側のみ保持
- フロントには露出しない
- 外部 API の raw レスポンスをそのまま表示しない

## 11.4 保守性
- 既存 `market.py` を肥大化させすぎない
- 新ページ専用処理はモジュール化
- ヘルプ文言は定数管理

## 14. 実装上の注意点

### 14.1 良い点
この repo はもともと
- 無料プラン制約
- シンボル上限
- REST フォールバック
- 静的ページ追加のしやすさ
を持っているので、今回の要件と整合しています。

### 14.2 注意点
一方で、そのまま流用すると初見ユーザーには少し難しいです。特に以下は調整が必要です。

- 接続状態やクレジット概念が分かりにくい
- 詳細項目が多く、最初に何を見るべきか不明
- `LM Studio` 系は初見向け必須機能ではない
- 既存 `/` はやや中上級者寄り

そのため、新ページでは
- 表示項目を減らす
- 説明を増やす
- 無料制約を明示する
のが適切です。

---

## 15. 推奨最終判断

実装方針としては、

**「既存の MarketDataHub と API 群を再利用しつつ、初見向けの薄い新規ページを追加する」**

のが最も合理的です。
既存 Monitor の全面改修より、工数・安全性・保守性のバランスが良いです。
