次営業日株価予測ML 実装仕様書 v0.1

1. 目的
本仕様書は、無料で取得可能な株式関連データのみを用いて、次営業日の株価変動を予測する機械学習システムを実装するための要件、データ仕様、学習仕様、検証仕様、運用仕様を定義する。
本システムの主目的は「翌営業日の終値そのもの」を直接当てることではなく、実運用に近い意思決定に使いやすい以下の目的変数を予測することである。
- 第一目的: 翌営業日の騰落方向（二値分類）
- 第二目的: 翌営業日の銘柄相対順位（ランキング）
- 参考目的: 翌営業日リターン（回帰）

2. 対象範囲
2.1 対象市場
- 主対象: 日本株（東証上場普通株）
- 拡張対象: 米国株（将来拡張。SEC/Alpha Vantage 等を用いたアダプタを別実装とする）

2.2 対象時間足
- 日足のみ
- 予測時点は営業日Dのデータ確定後
- 予測対象は営業日D+1の終値方向またはD+1における相対順位

2.3 対象外
- 秒足、分足、板情報、約定履歴
- 自動発注
- ポートフォリオ最適化の高度化
- 信用取引、先物、オプション
- ニュース本文の大規模言語モデル要約基盤の本格構築（初版では簡易特徴量化まで）

3. 前提方針
3.1 学習方針
- 主モデルは表形式データに強い勾配ブースティング木を採用する
- ベースラインとして線形モデルを必ず実装する
- 深層学習は第2フェーズ以降とし、初版の必須範囲から外す

3.2 目的変数方針
- 終値そのものの回帰は銘柄間スケール差が大きく不安定になりやすいため、初版の主目的にはしない
- 主目的は以下のいずれかとする
  a. y_cls: sign(close_{t+1}/close_t - 1)
  b. y_rank: 日付tにおける全銘柄の翌日リターン順位
  c. y_reg: close_{t+1}/close_t - 1

3.3 推論時点の厳密化
- 営業日Dの予測に利用してよい情報は「営業日Dの予測生成時点までに取得済み・確定済みのもの」に限定する
- 決算、開示、ニュース、配当等は公表時刻を保持し、営業日Dの引け後に公表されたものはD+1予測にのみ利用する
- 将来データの混入を厳格に禁止する

3.4 利用者向け表示方針
- 画面・レポートの通常表示は、相場や機械学習を理解している利用者にとって簡潔で読みやすいことを優先する
- 一方で、株や関連用語をあまり知らない利用者でも、主要な指標・操作・設定に付与された help マークや補足説明を開けば、「何をしているか」を大枠で理解できる程度の説明を提供する
- 専門用語そのものは消さず、必要に応じて短い平易語や用途説明を補助表示する
- 補足説明は学習教材レベルまで長文化せず、1〜3文程度で役割と判断観点が伝わる粒度を基本とする

4.1 日本株主要ソース

- 日本株向けの主要な無料APIであり、株価・財務・カレンダーを比較的統一的に取得しやすい

- 無料プランの履歴期間および遅延制約があるため、初版はその範囲で設計する

8. ユニバース定義
初版ユニバースは以下の条件で固定する。
- 東証上場普通株
- 上場後60営業日以上経過
- 予測日時点で売買停止継続ではない
- 直近20営業日で価格欠損が許容閾値未満
- 流動性フィルタを満たす（例: 20日平均売買代金が一定以上）

理由:
- IPO直後・低流動性・長期停止銘柄はノイズが大きく、初版学習を不安定化させるため

9. ラベル仕様
9.1 分類ラベル
y_cls_1d = 1 if return_1d > cost_buffer else 0
ここで
return_1d = adjusted_close_{t+1} / adjusted_close_t - 1
cost_buffer は取引コスト吸収のための閾値で、初期値 0.0 または 0.002 を設定可能とする

9.2 ランキングラベル
同一営業日tに属する全銘柄の return_1d を昇順または降順に並べ、分位点ラベルを付与する
例:
- 上位20%: 4
- 次の20%: 3
- 中央20%: 2
- 次の20%: 1
- 下位20%: 0

9.3 回帰ラベル
y_reg_1d = clip(return_1d, lower=-0.2, upper=0.2)
極端値の影響を抑えるためウィンズライジングを行う

10. 特徴量仕様
10.1 価格系特徴量
各銘柄について以下を生成する。
- 1, 2, 3, 5, 10, 20, 60営業日リターン
- 5, 10, 20, 60営業日移動平均乖離率
- 5, 10, 20営業日高値更新率、安値更新率
- 当日高安レンジ = (high - low) / close
- ギャップ率 = open / prev_close - 1
- ローリング標準偏差（5, 10, 20営業日）
- ボラティリティ比率（短期/長期）
- 出来高変化率、売買代金変化率
- 連騰連落日数
- 曜日、月初/月末フラグ

10.2 市場相対特徴量
- TOPIX対比超過リターン
- 業種平均対比超過リターン
- 市場全体下落日の逆行率
- 業種内順位（過去5/20日リターン順位）
- 市場ボラティリティ水準との相対値

10.3 ファンダメンタル特徴量
- 売上成長率
- 営業利益成長率
- EPS成長率
- 利益率
- 予想修正率
- 配当予想修正フラグ
- 直近決算サプライズ近似値（予想との差または前回予想からの変化）
- 時価総額近似カテゴリ（取得不可なら price×shares ではなく流動性帯で代替）

10.4 イベント特徴量
- 決算発表予定まで残り営業日数
- 決算発表後経過営業日数
- 直近EDINET提出からの経過営業日数
- form_type別フラグ（有報、四半期報告、大量保有、臨時報告など）
- 修正報告書フラグ
- 配当/分割イベント近傍フラグ

10.5 テキスト系簡易特徴量（任意）
- 開示タイトルのキーワードフラグ
- ニュース件数（当日/過去3日/過去7日）
- 平均センチメント
- センチメント分散
注記:
初版では文章埋め込みではなく集計特徴量までとする

10.6 カテゴリ特徴量
- sector17_code
- sector33_code
- market_segment
- month
- weekday
これらは one-hot 前提にせず、モデル側のカテゴリ処理を優先する

11. 欠損値処理仕様
- 価格系特徴量の計算に必要な履歴不足期間は欠損のまま保持し、学習時にモデルで処理する
- 財務値が未公表の期間は forward-fill を原則禁止する
- 財務イベント系は「未発生」と「欠損」を区別する
- カテゴリ欠損は '__MISSING__' で埋める
- 数値欠損は必要に応じてNULLのまま保持し、単純埋めは最後に限定して実施する

12. データリーク防止仕様
- adjusted_close_{t+1} を使うラベル作成処理は、特徴量作成処理と完全に分離する
- 将来の決算予定変更、後日訂正された財務値を過去時点に遡って使わない
- EDINET/ニュースは published_at ベースで利用可否を判定する
- 分割・配当調整は「予測時点で既知か」を明示する
- 学習データセット出力時に leakage_check を必須実行する

13. 学習タスク仕様
13.1 ベースライン
- Logistic Regression: y_cls_1d
- Ridge Regression: y_reg_1d
目的:
複雑モデルが最低限これらを上回ることを確認する

13.2 主モデル
- LightGBM Classifier: y_cls_1d
- LightGBM Ranker: y_rank
採用理由:
- 表形式時系列特徴量との相性がよい
- 欠損とカテゴリを扱いやすい
- 速度と性能のバランスがよい

13.3 代替モデル
- CatBoost Classifier / Ranker
用途:
- カテゴリ変数が多い場合
- ニュース見出しや開示タイトルをテキスト特徴として直接扱いたい場合

13.4 追加モデル
- XGBoost Ranker
用途:
- ランキング性能の比較
- 分位点予測や順位特化の比較実験

15. 検証仕様
15.1 分割方式
- 時系列の walk-forward 検証を採用する
- ランダム分割は禁止
- 各foldは train < gap < valid の順で並ぶこと

15.2 推奨例
- 学習期間: 12か月
- gap: 5営業日
- 検証期間: 1か月
- これを時系列に沿ってローリング

15.3 パネルデータ扱い
- 同一日付の全銘柄を同一foldに含める
- ある日付の一部銘柄だけ train、残りを valid に分けることは禁止

15.4 チューニング方針
- 初版は hand-tuned の固定パラメータで開始
- その後 Optuna 等で逐次最適化
- 最適化指標は分類ではICまたはAUC、ランキングではNDCG@k とバックテスト収益を併用

16. 評価指標
16.1 統計評価
分類:
- ROC-AUC
- PR-AUC
- balanced accuracy
- hit ratio
回帰:
- MAE
- RMSE
- Spearman IC
ランキング:
- NDCG@10
- Precision@10
- Top-decile spread

16.2 売買評価
単純な検証用ルールを固定する。
- 毎営業日、予測スコア上位N銘柄を翌営業日に1日保有
- 等金額配分
- 売買コスト控除あり
- 最大保有数、流動性フィルタあり
算出指標:
- CAGR
- Sharpe
- Max Drawdown
- Turnover
- 勝率
- 平均保有損益

17. バックテスト仕様
- シグナル生成はD日引け後
- 約定仮定はD+1日寄りまたはD+1日終値のいずれかを選択可能にする
- 初版は「D日引け後予測 → D+1日寄り買い → D+1日引け売り」を標準とする
- ストップ高/安や売買停止は約定不能フラグを立て、保守的に約定失敗として処理する
- 分割、上場廃止、データ欠損日の処理規則を明文化する

18. 再学習・運用仕様
18.1 日次バッチ
毎営業日、以下を順に実行する。
1) データ取得
2) 正規化
3) 特徴量更新
4) 推論
5) レポート出力
6) 結果保存

18.2 再学習頻度
- 初版は週次再学習
- 月次フルリビルドを別途実施
- モデル劣化時に臨時再学習を可能とする

18.3 モデル採用条件
新モデルは以下を満たす場合のみ本番採用する。
- 直近3fold平均で既存モデル以上
- バックテストの net 指標が既存モデル以上
- 特徴量数と計算時間が許容範囲内
- 異常値予測や欠損率上昇がない

20. 品質保証
20.1 テスト
- unit test: 変換関数、ラベル関数、特徴量関数
- integration test: API取得から学習データ生成まで
- regression test: 既知期間の評価値が大きく崩れていないか確認
- leakage test: 将来日付混入の検出

20.2 監視
- 取得件数異常
- 欠損率上昇
- 銘柄数急減
- 特徴量分布ドリフト
- 予測スコア分布ドリフト
- 実績損益ドリフト

21. 非機能要件
- 全日次バッチは1台の一般的な開発機でも完走できること
- 途中失敗時に再開可能であること
- 生データを再取得せず再現実行できること
- 学習結果は設定ファイルと乱数seed込みで再現可能であること
- ログはすべてタイムスタンプ付きで保存すること

22. 実装優先順位

- price/fundamental/earnings の正規化
- ラベル生成
- LightGBM classifier

Phase 2
- EDINET collector

Phase 3

- ニュース特徴量

23. 最低受入基準
- 学習データセットがリークなしで生成される
- 5fold相当の時系列検証が完走する
- ベースラインと主モデルの評価比較が出力される
- 日次予測ファイルが再現可能に出力される
- バックテスト結果がコスト込みで算出される

24. 初版の推奨結論
初版は以下で固定して着手する。
- 市場: 日本株
- データ: J-Quants + EDINET
- 目的変数: 翌日方向分類
- 主モデル: LightGBM Classifier
- 比較モデル: Logistic Regression
- 検証: walk-forward + gap
- 売買評価: 翌営業日寄り買い・引け売りの1日保有

25. 補足メモ
- 将来的に「翌日上がるか」より「翌日どの銘柄を優先するか」のほうが実運用と整合しやすいため、Phase 2以降で ranking を本命化する
- ニュース本文や開示本文の深い自然言語処理はコストに対して複雑度が高いため、初版では件数・センチメント・キーワードフラグに留める
- 無料データは遅延や欠損がありうるため、本番運用でもデータ欠損時のフォールバック設計を必須とする
